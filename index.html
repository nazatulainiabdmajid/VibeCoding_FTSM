Module 1

The Right Foundation & Vibe Coding Mindset

Module Objective:

Understand the definition of Vibe Coding from an AI expert, the UX design framework (ICQ), and the technical framework (B.A.S.I.C) for effective AI communication.

1.1 What is 'Vibe Coding'?

According to the Merriam-Webster Dictionary, vibe coding (also written as 'vibe-coding') is a recently coined term for the practice of writing code, making web pages, or creating apps simply by telling an AI program what you want and letting it create the product for you. In vibe coding, the user does not need to understand how or why the code works and must often accept that a certain number of bugs or glitches will be present.

The term was popularized by Andrej Karpathy (AI Researcher & Co-founder of OpenAI) in February 2025. He describes a workflow in which the user’s primary role shifts from 'writing code' to 'guiding an AI assistant'.

1.2 UX Steering Layer: The I.C.Q Framework

Vibe Coding produces functional artifacts quickly. This speed introduces risk. Without a clear UX framework, the output might lack structure. UX professionals use the Intent-Context-Quality (ICQ) model as a steering layer (Hohn & Loydl, 2026):

Intent

What is to be achieved? The professional objective for users, not technical realization. E.g., not "Build a dashboard", but "Provide an at-a-glance status overview."

Context

Under which conditions? Visual constraints, existing systems, accessibility. Concrete artifacts (like sketches) restrict AI interpretative variance.

Quality

How do we recognize a good result? Is it runnable? Understandable UX? Integrable? Vibe coding shifts effort from manual typing to quality curation.

1.3 Technical Framework: The Science of Prompting (B.A.S.I.C)

If ICQ is the 'What' (UX Philosophy), then B.A.S.I.C is the 'How' (The technical framework for writing text to Gemini).

[B] Boundaries: Set strict limits (e.g., "Build a tourism app", not "Build a world app").
[A] Actions: State the Input, Process, and Output clearly.
[S] Storage: Do you need temporary memory or a permanent database?
[I] Isolate: Build step-by-step. Don't ask for the whole app at once.
[C] Correct: Treat AI like an intern. If it makes a mistake, ask it to fix it!

I love the checkboxes! But when I refresh the app, my grocery list disappears. Please update the code to save the list permanently in the phone's local storage (using AsyncStorage) so I don't lose my data.

Copy the newest code, paste it back to App.js. Try adding items, refresh Expo Snack, and see that your list is now saved permanently in the phone!

Module 4

Native Power

Module Objective:

Understand web platform limits, transition to native Android Studio for hardware/sensor integration.

Introduction: Web Limitations

In Module 3, we built the Expiry Tracker manually. But we are lazy to type various date formats! We want to use the camera to scan expiry dates offline!

Limitation: Web Apps (Expo) struggle with heavy offline Machine Learning (OCR). For smart camera sensors, we must use native power: Android Studio.

Hands-On Activity 6: The Native ML Prompt (Smart Camera)

1

Open Android Studio on your computer. Open the built-in Gemini Agent in the right panel.

2

Send this 'Mega' B.A.S.I.C prompt to the Agent. Notice how we ask it to build a 'target box', save the product name, build a list, and automatically calculate the days left until expired!

Act as a Senior Android Developer. Create an Expiry Date Tracker app for leftover food using Kotlin and Jetpack Compose (Material 3). [B] The app allows users to track food items by scanning expiry dates using the camera or entering them manually. [A] UI & Functional Logic Requirements: • On-Demand Scanner: Show a dark placeholder box with an 'Open Scanner' button. When clicked, launch a CameraX Preview. Overlay a Targeting Box (bounding box) in the center. Include a manual close ('X') button to stop the camera and save battery. • Input Fields: An OutlinedTextField for Expiry Date (auto-filled by scan or manual entry), and Product Name. A 'Save Item' button enabled only when both are filled. • Scan Logic: Use ML Kit to recognize text ONLY inside the center targeting box. Use Regex to find date patterns (e.g., DD/MM/YYYY, MM/YY). Auto-close the camera once a date is detected and focus on the Product Name. • Saved List: Below the inputs, display a list of cards showing Product Name, Expiry Date, and dynamically calculate 'Expires in X days'. Highlight the text in Red if <= 3 days. Clear inputs immediately after a successful 'Save'. [S] Use temporary state memory for now. [I] Tech Stack & Permissions: • CameraX (Core, Camera2, Lifecycle, View) & Google ML Kit Text Recognition. Use a LazyColumn for the screen. • Handle CAMERA permission gracefully within the UI. Include the necessary <uses-permission> and <uses-feature> for the manifest. • Provide the complete working code in one go.
3

The Agent will generate the Kotlin code and update your MainActivity.kt file live. You no longer need to copy and paste manually!

4

Press the Run (▶️ Green icon) button at the top to launch the app on the Emulator!

Frequently Asked Questions

I'm confused. Should I use Replit, Expo, or Android Studio?
Use Replit for the fastest visual web apps. Use Expo Snack for medium mobile logic. Use Android Studio if your project needs heavy hardware like cameras or background notifications.
Android Studio makes my laptop lag. What should I do?
If your laptop is struggling, do not force it. This is exactly why we use Replit & Expo Snack! Start your prototype on the web because it uses zero laptop memory. Once your logic is perfect, borrow a lab computer just to compile Android Studio.
If I use Gemini for my Final Year Project (FYP), is it cheating?
Using AI is not cheating if you use it to generate the first draft of your code. For computing students, you must customize the code yourself using the programming language. For non-computing students, please refer to your lecturer for the allowed guidelines.
